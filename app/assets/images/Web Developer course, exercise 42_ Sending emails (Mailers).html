    Learning Goals

    • Create a contact form using the form_tag helper with a variety of form elements, such as text fields, checkboxes, and radio buttons
    • Change HTTP method of form_output route from GET to POST and explain why
    • Use form data to send an automated email using the ActionMailer method in development environment
    • Use Sendgrid plugin on Heroku to send automated emails from production environment
    • Professional level: Customize email templates by generating Mailers and their corresponding Views

    Introduction

    We’ve been introduced to two different types of forms and different types of HTTP Requests. We used the GET method in our search box. In this lesson, we’ll use more form_tag helpers to POST a contact form to your application, which will send out an automated email. For this purpose Rails includes an email library called Action Mailer. Action Mailer is similar to controllers: you will create a method and corresponding view template for each type of email that your application sends out.

    The Contact Form

    For this exercise, we will add a form to our contact page. When a user submits this form, it will POST the form data to a new action called thank_you and that will send an email to your email address (as the “admin” of the site) with the contact message. The minimum fields we need in the form are a message field and a name and email so we can respond to the user’s message appropriately.

    <h1>Contact Us</h1>

<%= flash[:notice] %>

<%= form_tag("/static_pages/thank_you") do %>
<div class="row">
  <div class="col-md-4">
    <div class="form-group">
      <%= label_tag :name, "Name" %>
      <%= text_field_tag :name, nil, class: "form-control", placeholder: "Enter Name" %>
    </div>      
    <div class="form-group">
      <%= label_tag :email, "Email Address" %>
      <%= text_field_tag :email, nil, class: "form-control", placeholder: "Enter Email" %>
    </div>
    <div class="form-group">
      <%= label_tag :message, "Message" %>
      <%= text_area_tag :message, nil, class: "form-control", placeholder: "Enter Message" %>
    </div>      
  </div>
</div>
<%= submit_tag "Send" %>   
<% end %>

    And your form should look something like this (if you haven’t styled any of your site yet):

    If you try submitting your contact form right now, you’ll get a missing route error.

    This is because our thank_you action doesn’t have a route yet. Let’s create one in routes.rb to accept our POST form data:

    post 'static_pages/thank_you'

    If you try to submit the form now you’ll get a missing template error, so you need to create one for the thank_you action. You can add any message you want but here is the placeholder we’ve used for ours (in app/views/static_pages/thank_you.html.erb):

    <div class="panel panel-default">
  <div class="panel-body">
    <p>Thank you. We have received your message and will respond as soon as possible.</p>
  </div><!-- / panel-body -->
</div><!-- / panel -->

    Feel free to style this page differently. Add an image or a link back to the homepage.

    Now you can submit the form without any problem, so now we need to take care of the back end component.

    Send emails in Developer Mode

    When the form gets submitted, we will retrieve the form data using the params hash like we did before; only this time we will use that data to send an email to our admin user (you). The simplest way to send an email from the controller is using the mail method which is available to us in the ActionMailer::Base class. We can call this directly from within our controller action and pass it the parameters from the email form:

    ActionMailer::Base.mail(:from => 'from@domain.com', :to => 'to@domain.com', :subject => "Welcome to My Awesome Site", :body => 'I am the email body.').deliver_now

    deliver_now sends emails immediately. deliver_later is used to deliver your email through an Active Job (Check out the Rails Guide if you are interested in Active Jobs)

    You can add this method to your controller method, replacing each of those values with the values from the params we received from the form.

    def thank_you
  @name = params[:name]
  @email = params[:email]
  @message = params[:message]
  ActionMailer::Base.mail(:from => @email,
      :to => 'your-email@example.com',
      :subject => "A new contact form message from #{@name}",
      :body => @message).deliver_now
end

    Here we are using the user’s email address as the “from” field, you will enter your own email address for the “to” field, and the body will just be a copy of the message they entered in the form.

    In development mode, when you send an email, the email doesn't actually get sent anywhere, it just gets printed out in the logs where "rails server" is running. To check whether the emails you are sending are working correctly, just check the terminal logs for a message that look like this:

    ActionMailer::Base#mail: processed outbound mail in 13.4ms

Sent mail to to@domain.com (18.0ms)

From: email@email.com

    Send emails from Heroku

    In order to send real emails, you need to configure your app to use a mail server. You should use a third party email provider so you don’t need to set up and maintain an email server. For your app on Heroku, we recommend using the Sendgrid plugin, a free, cloud-based email provider. On Heroku, in order to install their plugins you need to link a credit card to your account. Don’t worry, Sendgrid (like most Heroku plugins) has a free starter plan and your credit card won’t be charged unless you upgrade your plan.

    To install the Heroku addon, you can simply run the following code in terminal in your app folder:

    heroku addons:create sendgrid

    Alternatively, you can go to your app dashboard, select your app and click the “get more add-ons” link on the “Resources” tab. If you haven’t already linked your credit card to your account, you will be given instructions to do so.

    Once you have the plugin installed it will appear listed on your app dashboard at heroku.com. Go to your config/environment.rb file and add the following lines to the bottom. This will enable your ActionMailer to use the Sendgrid plugin to send out emails from Heroku:

    ActionMailer::Base.smtp_settings = {
  :address => 'smtp.sendgrid.net',
  :port => '587',
  :authentication => :plain,
  :user_name => ENV['SENDGRID_USERNAME'],
  :password => ENV['SENDGRID_PASSWORD'],
  :domain => 'heroku.com',
  :enable_starttls_auto => true
}

    Now launch your application to Heroku and try it out!

    Generate HTML Emails

    In order to send HTML-formatted mail that can be styled, you need to create a template and a Mailer. Mailers are the classes that handle the email logic for your app. You don’t have any right now, so let’s create one using the rails generate command in terminal:

    $ rails generate mailer UserMailer

create  app/mailers/user_mailer.rb
create  app/mailers/application_mailer.rb
invoke  erb
create    app/views/user_mailer
create    app/views/layouts/mailer.text.erb
create    app/views/layouts/mailer.html.erb
invoke  test_unit
create    test/mailers/user_mailer_test.rb
create    test/mailers/previews/user_mailer_preview.rb

    If you take a look at app/mailers/application_mailer.rb, you'll see:

    class ApplicationMailer < ActionMailer::Base
  default from: "from@example.com"
  layout 'mailer'
end

    This is where you will define the default email that will show up in the “from” field of emails that get sent from your application. Go ahead and exchange form@example.com with the email address you want to use on your app.

    Now open app/mailers/user_mailer.rb. You’ll see:

    class UserMailer < ApplicationMailer
end

    Inside of the UserMailer class, you will define any of the emails your application will send to users. Let’s start by adding in the contact_form email method, which is essentially just a copy of the same method from the controller:

    class UserMailer < ApplicationMailer
  default from: "from@example.com"

  def contact_form(email, name, message)
  @message = message
    mail(:from => email,
        :to => 'your-email@example.com',
        :subject => "A new contact form message from #{name}")
  end
end

    However, the mailer class won’t have access to the variables in our controller action (@email, @name and @message) so we will have to pass those into the mailer method as parameters. We will also be using the message inside the mail template, so we set that to an instance variable here. And we don’t need to set the "body" parameter of the mail message because we are using a template for the body.

    We then call the deliver_now method from inside our controller action to fire off the email:

    def thank_you
  @name = params[:name]
  @email = params[:email]
  @message = params[:message]
  UserMailer.contact_form(@email, @name, @message).deliver_now
end

    Now there’s one last step which is to create the view, which will be an HTML template that gets sent as the email. Our email template will be called contact_form.html.erb (it needs to match the name of the mailer method), and it will go in app/views/user_mailer. These behave just like any of your other views: you can insert ruby tags, instance variables, and render partials.

    However, HTML email templates require slightly different considerations. Best practices dictate that you should always use HTML tables for the layout of the email so they render properly in as many email programs as possible, and you should always use inline CSS styles for any styling. We will start you off with a very basic template with no styling, but check the next section for tips on how to customize them:

    <table>
  <tbody>
    <tr>
      <td><p>You have received a new contact form message. The contents of the message are:</p></td>
    </tr>
    <tr>
      <td><p><%= @message %></p></td>
    </tr>
  </tbody>
</table>

    Preview Email Templates

    Since Rails 4.1 you are able to preview your email templates locally while you are styling them. Fortunately, Rails makes this very easy for us. When you generated the UserMailer before, Rails automatically created a couple files inside the test/mailers/previews folder of your app. (If you followed the instructions of this course to generate your Rails app, you probably generated your app without a test folder. In that case you simply create the a folder called test inside your project directory, now. Then inside test, add another folder called mailers. And inside that folder you create a previews folder.)
    In that last folder, there should be a file called user_mailer_preview.rb and if you open it you see this:

    # Preview all emails at http://localhost:3000/rails/mailers/user_mailer
class UserMailerPreview < ActionMailer::Preview

end

    Inside that class, you can call the same method we defined earlier inside the app/mailers/user_mailer.rb file: contact_form. Then inside that method you call UserMailer.contact_form() and add values for the three arguments we defined earlier: email, name, message.
    An example could look like this:

    # Preview all emails at http://localhost:3000/rails/mailers/user_mailer
class UserMailerPreview < ActionMailer::Preview
  def contact_form
    UserMailer.contact_form("john@example.com", "John", "Hello World!")
  end
end

    Now, while your server is running locally you can preview the email following the link in the comment: http://localhost:3000/rails/mailers/user_mailer
    There you click on "contact_form" and Rails will render the email for you. Alternatively you can go straight to http://localhost:3000/rails/mailers/user_mailer/contact_form

    Professional Level: Customize HTML Email Template

    For more info on creating beautiful HTML emails, tutsplus has a great tutorial on designing one from scratch. Mailchimp has a great resource on all things email-- best practices, layouts, images, calls to action. This is a great read if your potential company or client is going to use email campaigns!
